# Selenium E2E сценарии

Минимальные сценарии, покрывающие ключевые требования:

1) Студент без авторизации
   - открыть `BASE_URL`
   - найти группу (например, 121)
   - открыть расписание, добавить в избранное, обновить страницу и убедиться, что избранное сохранилось

2) Авторизация и управление пользователями
   - открыть модалку входа, залогиниться `superadmin@campusflow.com / admin123` (или переменные окружения)
   - перейти на `/admin`
   - открыть таб пользователей, отредактировать пользователя, удалить пользователя (с подтверждением)

3) Управление группами/подгруппами
   - открыть таб групп
   - создать/отредактировать группу
   - добавить подгруппу внутри модалки группы, обновить и убедиться, что подгруппа отображается

4) Расписание и конфликты
   - в табе расписания добавить пару для подгруппы
   - попытаться добавить пересекающуюся пару и получить предупреждение/ошибку (HTTP 409)

5) Роутинг
   - навигация на `/schedule/group/{num}` и `/schedule/teacher/{name}` отображает корректные данные
   - переход на `/admin` открывает панель администратора

Запуск примера: `node tests/selenium/basic.e2e.js` (нужен установленный `selenium-webdriver` и поднятый фронт).
